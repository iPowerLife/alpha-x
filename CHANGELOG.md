## 2026-06-17 — Задача #3: каркас нового приложения ✅

**Что сделали:** с нуля собрали скелет нового приложения и доказали, что вся
цепочка работает на настоящих данных.

| Шаг | Результат |
|-----|-----------|
| Чистый Next.js 16 в папке `alphamarket-ts` | App Router, готов |
| Старую папку с миграцией переименовали в `alphamarket-mig` | архив |
| Подключили Prisma (мост к базе) | видит 74 модели/таблицы |
| Подключили клиенты Supabase (браузер/сервер/middleware) | через навык `supabase` |
| Сделали первый эндпоинт `/api/countries` | вернул реальные страны из базы за 0.84с |

**Как именно (способ):**
- Сгенерировали проект `create-next-app` (Next 16.2.9, React 19, TypeScript, Tailwind).
- Prisma 6 + `@prisma/client`; синглтон клиента в `src/lib/prisma.ts`, чтобы при
  горячей перезагрузке не плодились соединения.
- Клиенты Supabase разложили в `src/lib/supabase/` (client / server / middleware),
  корневой `src/middleware.ts` обновляет сессию.
- Проверили: `npm run dev` → запрос к `/api/countries` → 200 OK, 12 активных стран
  из 237, названия на русском/английском/сербском. Значит фильтр и данные на месте.

**🪤 Главная грабля — подключение к базе:**
- «Transaction pooler» (порт **6543**) с твоей локальной сети **не работает** —
  Prisma висит 20 секунд и рвёт связь (ошибка P1017). Это особенность твоей сети.
- «Session pooler» (порт **5432**) **работает** стабильно (~1 секунда).
- **Решение:** локально в `.env` используем **5432**. Для будущего деплоя на Vercel
  в `.env` оставлена закомментированная строка с 6543 (там сеть другая, 6543 норм).

**Заметка про ключи:** ключи Supabase в `.env` пока заглушки (`ЗАМЕНИТЬ_...`).
Нужны будут позже (вход/файлы) — возьмёшь в дашборде.

📌 Память: `project-app-scaffold.md`, `project-status.md`.

## 2026-06-17 — Задача #2: перенос базы данных в Supabase ✅

**Что сделали:** перелили все данные старого маркета в облачную базу Supabase
(Postgres). Ты подтвердил «Success» в дашборде.

**Как именно (способ):**
- Развернули схему: **74 таблицы** в схеме `public`.
- Залили данные скриптом-загрузчиком (`alphamarket-mig/load_data.cjs`): на каждую
  таблицу своя транзакция, на время заливки отключали проверку связей между
  таблицами, грузили пачками по 200 строк, потом поправили счётчики ID.
- По дороге починили 3 ошибки заливки: несовпадение типов (привели к TEXT),
  и две ошибки со счётчиками (`setval`).
- По твоей просьбе вычистили 27 картинок, вшитых прямо в базу (base64) — их позже
  положим в нормальное файловое хранилище (Storage).

**Важно на будущее (авторизация):** старые пользователи лежат в таблице
`public.testsite_users` и **никак не связаны** со встроенным входом Supabase (Auth).
Когда дойдём до авторизации (особенно входа через Telegram) — отдельно решим,
как их «подружить».

📌 Память: `project-db-migration.md`.
