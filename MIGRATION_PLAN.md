## 🚀 Финальный стек (решено)
| Сейчас (Python) | Станет (TypeScript) |
|---|---|
| Django (сервер) | **Next.js на Vercel** (фронт + API в одном приложении) |
| HTML-шаблоны Django | **React-страницы (Next.js)** |
| MySQL | **Supabase (PostgreSQL)** — потребует конвертацию дампа MySQL→Postgres |
| Django ORM | **Prisma** (решено — берём) + supabase-js для auth/realtime/storage |
| Django auth + JWT | **Supabase Auth** (+ мостик для входа через Telegram) |
| Django Channels (WS/чат) | **Supabase Realtime** |
| Медиа-файлы | **Supabase Storage** |
| Celery (обновление курсов) | **Vercel Cron** |

Отброшено по ходу: NestJS (не нужен — бэкенд живёт внутри Next.js на Vercel), BullMQ (не нужен — задача по расписанию решается Vercel Cron).

## ⚠️ Ключевые риски (держать в голове)
1. **Финансовая логика = деньги.** Сделки/комиссии/балансы/резервы переносим по частям с проверкой паритета (новый код считает точь-в-точь как старый Python). Не big-bang.
2. **MySQL → PostgreSQL.** Дамп надо один раз сконвертировать.
3. **Telegram-авторизация.** Нужен мостик: проверка данных от Телеграма → сессия Supabase.
4. **Парсинг курсов на Vercel.** Лимит времени serverless-функций. Если не уложится — вынести в отдельный мелкий воркер. Проверить на этапе карты.
